# Platform-to-Event Mapping

## How to Identify Platform in Amplitude

The `platform` property is automatically set by the SDK:
- **Web** ‚Äî Segment JS SDK from galileo-site (desktop browsers)
- **Mobile Web** ‚Äî Segment JS SDK from galileo-site (mobile browsers, detected by user-agent)
- **iOS** ‚Äî Segment iOS SDK / Firebase from xe-apollo
- **Android** ‚Äî Segment Android SDK / Firebase from xe-apollo

## Codebase-to-Platform Mapping

| Codebase | Platforms | Analytics Pipeline |
|----------|----------|-------------------|
| galileo-site | Web, Mobile Web | Segment JS ‚Üí Amplitude (295336) |
| xe-apollo | iOS, Android | Segment Mobile + Firebase + AppsFlyer ‚Üí Amplitude (295336 + 295341) |

## Event Source by Platform

### Web + Mobile Web Only (galileo-site)
These events are exclusively fired from the web codebase:
- All `SEGMENT_EVENTS` constants in `galileo-site/src/constants/segmentAnalytics.ts`
- Page view events via `analyticsStore.page()`
- GTM events via `analyticsStore.gtmTrack()`
- LogRocket session tracking

### iOS + Android Only (xe-apollo)
These events come from the mobile app:
- `Application Installed` (AppsFlyer)
- `Application Opened` (Firebase/Segment)
- `Install Attributed` (AppsFlyer)
- `af_mt_profile_created` (AppsFlyer)
- Push notification events (`[Iterable] pushSend`)
- Mobile-specific biometric events

### Cross-Platform Events (Both Codebases)
These events are fired from both web and mobile, tracked under the same name:
- `Registration Started` / `Registration Completed`
- `Login Started` / `Login Completed` / `Login Failed`
- `Quote Accessed` / `Quote Created` / `Quote Confirmed`
- `Transaction Created`
- `Payment Method Selected`
- `Recipient Created` / `Recipient Selected`
- `Profile Created` / `Profile Updated`
- `Biometric Verification Started/Completed`

### Server-Side Events (No Codebase)
These are generated by backend systems:
- `Account Approved` / `Account Approved with STP`
- `Account Closed` / `Account Restricted`
- `Fraud Check Completed/Referred/Started`
- `PEP SAN Check Completed/Started`
- `Risk Assessment Completed`
- `Disbursement Sent`
- `Payment Completed`
- `Transaction Status Updated`
- `eKYC Results Received`
- `Scheduled Payment Failed/Reminder`

### Third-Party Events
- `[Iterable] emailBounce/emailSend/emailSubscribe/inAppSend/pushSend` ‚Äî Email/push from Iterable
- `[Amplitude] Page Viewed` ‚Äî Auto-tracked by Amplitude
- `XECD *` events ‚Äî From the XE Currency Data product (separate business line)

## Key Views and Their Events (galileo-site)

### SendMoney Flow (`src/views/SendMoney/`)
| View | Key Events |
|------|-----------|
| SendMoneyPayment.vue | Payment Method Selected, Card Auth Started/Failed, Open Banking events, Bank Transfer events, Transaction Summary Confirmed |
| SendMoneyVoltInfo.vue | Open Banking Info Page Displayed |

### QuickTransfer (`src/views/QuickTransfer.vue`)
- Quick Transfer Accessed, Quote Created, Quote Confirmed, Quick Transfer Completed

### Login (`src/views/Login.vue`)
- Login Started, Login Failed, Login Completed

### Recipients (`src/views/Recipients.vue`)
- Recipients Accessed, Recipient Created, Recipient Selected, Recipient Creation Started, Recipient Info Added, Recipient Bank Details Added

### Dashboard (`src/views/Dashboard.vue`)
- Dashboard page view, various feature access events

### Account (`src/views/Account/`, `src/views/AccountSettings/`)
- Settings Accessed, Change Email/Password/Phone Started, Profile Updated

### PlaidOauth (`src/views/PlaidOauth.vue`)
- Bank Account Selection Started/Completed, Bank Verification Completed

### Corporate (`src/corporate/`)
- Auth Signatory Submitted, Multiple Payments events, User Roles events, Convert Balance events, Corporate KYC refresh events
