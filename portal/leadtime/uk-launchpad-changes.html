Rows are sorted by payment volume. CCY Type = Ria CCY means the Ria row won over the Standard row in the current CSV (launchpad behaviour). IN = Mon–Fri before 14:00 GMT, OUT = everything else. E2E (L1+L2) adds the bank-to-bank Leg 1 median (397s) to the Leg 2 P80 — that\'s the number the proposed tier is based on.
These currencies have two rows in UK.csv. The launchpad loader keeps only one per currency — the Ria CCY row wins; the other row becomes dormant. See the Row-Selection Logic tab for the exact code.
When UK.csv contains multiple rows for the same Currency, the launchpad loader collapses them to one. Here is what actually runs.
Step 1 — Detect which row is the Ria variant
// ParseLine: for each CSV row, decide if it's a Ria row by looking at column 13 (the "CCY" column).
// c[13] is the CCY cell value: "Major CCY", "Standard CCY", "Exotic CCY", or "Ria CCY".
return Traverse(reads).Map(c =>
    new LeadTimeEntry(
        c[0]?.Trim(),
        // ...
        c[13].Contains("Ria", StringComparison.InvariantCultureIgnoreCase)  // IsRia
    ));
Step 2 — Collapse rows, Ria wins
// ToMap: group all entries by (Currency, DestinationCountry), then pick one per group.
// If any row in the group is Ria, take it. Otherwise take the first row.
entries
    .GroupBy(e => (Currency: e.Currency, Country: e.DestinationCountry), ...)
    .ForEach(g =>
    {
        map[key] = g.FirstOrDefault(x => x.IsRia) ?? g.First();
        //            ^^^^^^^^^^^^^^^^^^^^^^^^^ Ria wins if it exists, else first row
    });
Concrete example — INR
UK.csv has two rows for INR. The Ria row wins, so customers sending GBP → INR see “Typically minutes”, not “Up to 3 days”.
  • Row 1: INR, <3 days, ..., Exotic CCY, ... — Dormant (loses to Ria)
  • Row 2: INR, 5mins, ..., Ria CCY, ... — Active (IsRia=true, wins)
  • Loader result: map["INR"] = Row 2
  • Quote response: q.LeadTime = "Typically minutes"
What this means for our changes
When we compare a proposed SLA against the “current” SLA, we replicate this same Ria-priority logic — so the “current” column always shows what a customer actually sees today (the active row), not just the first line we find in the file. Our new per-destination rows inherit the active row’s values for anything they don’t override.
